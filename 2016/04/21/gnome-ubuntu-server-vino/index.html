ubuntu_server_vino

ubuntu server在安装桌面环境gnome-shell后便会自带远程桌面共享（也就是vino）的功能，但有时候因为桌面环境的安装问题，可能会找不到设置的界面。
在终端中运行命令：

vino-preferences

如果提示未找到命令，则按提示 sudo apt-get install vino 安装下vino

在弹出的图形化界面中设置好参数后，用vnc连接却报错，如图：

看意思是说不支持数据传输加密方式啥的，google下，找了一会，解决方案就是把服务器那边的传输方式设成不加密…

终端执行命令:

gsettings set org.gnome.Vino require-encryption false
